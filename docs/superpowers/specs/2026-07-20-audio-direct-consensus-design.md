# 오디오 직접 청취 + 교차 합의 재설계 (v3)

작성일: 2026-07-20
상태: 설계 확정 (구현 전) · 실측 벤치마크 근거 포함

## 1. 배경과 문제 정의

이 도구의 존재 이유: **한국인 검수자는 영어 로컬라이제이션 품질을 스스로 판단할 수
없다.** "이 영어 더빙이 영어권 시청자에게 제대로 전달되는가"를 확인해줄 원어민
감각이 사내에 없으므로 AI가 그 역할을 대신해야 하고, 따라서 사람이 AI의 지적을
검증한다는 전제가 성립하지 않는다. **AI 판단이 자체적으로 신뢰 가능해야 한다.**

현재 구조(v1/v2 스펙)의 근본 문제 두 가지:

1. **STT가 부정확하다.** 한국어 SRT도 대본도 없어 한국어 원문은 STT에 전적으로
   의존하는데, Whisper 계열(ko-turbo 포함)의 정확도가 실무 수준에 못 미친다.
   Demucs 보컬 분리를 붙여도 개선되지 않았다(실측). 원문 텍스트가 틀리면 정렬,
   대조 판단, 점수 전부가 오염된다.
2. **오탐이 많다.** "문제를 찾아라" 프롬프트를 받은 페르소나는 뭐라도 찾아내려
   하고(LLM judge의 고질적 편향), 같은 Gemini 하나가 역할극 3번 한 것끼리의
   "동의"는 증거력이 약하며, 지적이 검증 없이 감점으로 직결된다. 검수자가 오탐을
   판별해줄 수 없는 도구 특성상, 못 믿을 지적은 없느니만 못하다.

성공 기준: 판정(점수 자체보다 평가 요소별 판단)이 사내 보고·의사결정에 쓸 수
있을 만큼 신뢰 가능할 것. 원칙은 **"확실한 것만, 적은 건수로"** (정밀도 우선).

## 2. 핵심 설계 결정

### 결정 1 — 한국어를 텍스트화하지 않고 오디오로 직접 듣는다 (STT 우회)

- 영어 SRT(사람이 만든 확정 자료)를 타임라인 기준으로 고정한다.
- 씬 단위로 한국어 원본 오디오를 잘라, 판정 모델이 **직접 청취**하며 영어 더빙과
  대조한다. 판단에 필요한 것은 받아쓰기가 아니라 의미·뉘앙스 파악이므로 소음에
  더 강건하고, 못 들은 구간은 환각으로 채우는 대신 "판단 보류"로 분류시킬 수 있다.
- STT는 판단 경로에서 제거하고 화면 표시·참고용으로만 유지한다. STT가 틀려도
  판정은 오염되지 않는다. Demucs 보컬 분리는 청취 경로에 넣지 않는다(§4 실측).

### 결정 2 — 확정은 독립 2회 청취의 "의미 수준 합의"로 (오탐 필터)

- 검출층이 낸 후보 지적마다, 같은 씬 클립을 **다른 모델에 블라인드로**(첫 번째
  모델이 뭘 지적했는지 알려주지 않고) 다시 들려 독립 재평가한다.
- 두 판단이 **의미 수준에서 합의**한 것만 최종 결과에 올린다(자구 일치가 아님 —
  §4 실측에서 자구 일치 기준은 커버리지를 43%로 떨어뜨리고, 의미 기준은 정밀도
  손실 없이 64%+로 올림).
- 불일치 → 폐기. 청취 불가 → "판단 보류" 목록으로 정직하게 표시.
- 의미 일치 여부의 판정은 별도의 경량 텍스트 호출(두 판단 결과를 넣고 "같은
  문제를 같은 근거로 지적하는가"를 분류)로 수행한다 — 오디오가 없는 싼 호출이라
  비용 영향은 무시 가능.
- 당장은 Gemini 내 다른 모델로 시작하고, OpenAI 키 확보 시 GPT-4o 계열로 교체해
  계열 간 독립성을 확보한다(같은 계열 형제보다 상관 오류가 적을 것으로 기대).
- 합의 필터는 옵션이 아니라 **필수 부품**이다 — 단독 모델 정확도는 60% 수준(실측).

### 결정 3 — 운영 원칙

- **API 요청 하나에는 씬 하나만.** 여러 씬(오디오 여러 개)을 한 요청에 묶으면
  모델이 클립 귀속을 놓쳐 다른 구간의 대사를 갖다 붙인다(실측 2회 확인 — 배치
  혼동, 타임코드 어긋남).
- 씬 클립에는 **앞 맥락(±25초 수준)을 포함**한다. 맥락 없는 초단문 클립은 같은
  오류를 두 모델이 공유하는 상관 오류를 낳지만, 맥락을 주면 틀릴 때 서로 다르게
  틀려 합의 필터가 작동한다(실측).
- 모든 지적에 "모델이 들은 한국어 내용"을 병기한다 — 영어 판단은 검증 불가해도
  한국어 청취는 한국인 검수자가 즉시 검증 가능하다는 신뢰의 연결고리.
- 429(쿼터) 백오프·재시도를 프로바이더에 내장한다.

## 3. 파이프라인 (v3)

```
영어 SRT (필수) ──→ 타임라인 기준 고정, 침묵 간격 + 크기 상한으로 씬 분할
한국어 원본 오디오 ──→ 씬별 클립 추출 (맥락 포함, STT 없음)
영어 더빙 스템 ──→ 씬별 클립 추출

[검출층 — 영화 전체, 씬당 1요청]
  ├─ 문화전문가: 한국어 오디오 청취 + 영어 SRT + 지식베이스
  ├─ 원어민 시청자: 영어 SRT만 (한국어 안 봄 — 기존 그대로)
  ├─ 연출가: 한국어 오디오 + 영어 더빙 오디오 양쪽 청취
  └─ 룰 체크: 싱크 격차·발화속도·클리핑/SNR (결정론적, 그대로)
        ↓ 후보 지적들
[확정층 — 후보 씬만]
  다른 모델 블라인드 재청취 → 의미 수준 합의 판정
        ↓
[최종 출력]
  ├─ 확정 지적 + 수정안 (합의 수준, 들은 한국어 병기)
  ├─ 판단 보류 구간 (청취 불가/불일치 — 커버리지 정직하게 표시)
  └─ 룰 체크 결과
```

유지: 3-페르소나 역할·입력 구분, 지식베이스 YAML, 룰 체크, 피드백 JSONL,
페르소나별 대안 병렬 제시. 폐기·격하: STT 필수 경로(표시용으로 격하), STT 단어
매칭 정렬(씬 클립 추출로 대체), 페르소나 간 동의 승격(타 모델 합의로 대체).

## 4. 실측 벤치마크 근거 (2026-07-20, GhostSweepers 원본 10분, 25구간)

방법: 영어 SRT 타임코드로 한국어 원본에서 25개 발화 클립 추출, 각 조건에서
모델이 받아쓴 내용을 검수자(한국어 화자)가 직접 청취 채점(정확/부분/오류).

| 조건 | 정확 | 부분 | 오류 |
|---|---|---|---|
| Gemini flash-preview, 맥락 포함 (14개만 채점 — 쿼터) | 43% | 43% | 14% |
| Gemini flash-lite, 맥락 포함 | **60%** | 16% | 24% |
| 로컬 Whisper ko-turbo (현재 파이프라인 조건) | **24%** | 56% | 20% |

핵심 발견:

1. **Gemini 청취 > STT**: 완전한 문장 확보율 60% vs 24%. STT는 대부분 조각난
   부분 정답("그 땡중놈도 찾아서 데려오고" → "그 땡이랑"). 로컬라이제이션 판단엔
   온전한 문장이 필요하므로 결정적 차이.
2. **합의 필터 정밀도**: 두 모델 받아쓰기가 완전 일치한 6구간 = 채점 결과 전부
   "정확"(100%). 맥락 없는 1차 테스트에서 발생한 상관 오류 2건은, 맥락 제공 +
   1클립/요청 조건에서 불일치로 갈라져 필터에 걸렸다.
3. **커버리지**: 자구 완전 일치 기준 43% → 의미 수준 일치 기준 64%+. 합의 판정은
   의미 수준으로 한다.
4. **비명·감탄사·강소음 구간은 모든 방법이 실패** — 대사 로컬라이제이션 판단
   대상이 아니므로 "판단 보류"로 표시하면 충분하다.
5. **Demucs 보컬 분리는 청취에도 무효**: 25구간 중 확실한 개선 1건, 퇴보 1건,
   나머지 동일. 잘 들리는 건 분리 없이도 잘 들리고, 안 들리는 건 분리해도 안
   들린다. 편당 수십 분의 연산 비용 대비 효과 없음 → 청취 경로에서 제외.

## 5. 비용·쿼터

- 오디오 입력은 텍스트보다 같은 내용 기준 15~20배 비싸다(Gemini Flash: 텍스트
  $0.30/M vs 오디오 $1.00/M, 초당 ~32토큰). 그래도 절대 금액은 작다:
  2시간 영화 전체 청취 1회 ≈ 23만 토큰 ≈ 320원.
- 편당 추정: 검출층(문화전문가 1회분 + 연출가 2회분, 컨텍스트 캐싱 활용) ~800원
  + 확정층(후보만) ~300–500원 = **편당 약 1,200–1,500원** (현재 대비 2~3배이나
  수용 가능).
- **무료 티어로는 정기 운용 불가**: flash 계열 하루 20요청(실측) vs 새 설계는
  편당 300~500요청(씬당 1요청 원칙). 실무 운용은 유료(pay-as-you-go) 전제.
- **무료 저속 모드 (파일럿용)**: 쿼터는 모델별 별도이므로, 모델 풀(lite → preview
  → 2.0 계열)을 로테이션하는 폴백을 프로바이더에 내장하면 무료 티어로도 밤새
  배치로 편당 처리가 가능하다. 검출/확정에 서로 다른 모델을 쓰는 합의 구조와도
  맞물린다. 단 RPM 제한으로 느리고, 씬별 판정 모델이 달라져 품질이 불균일하며,
  무료 쿼터 정책 변경에 취약하므로 정기 운용은 유료로 한다. 429 백오프 + 모델 풀
  폴백 코드는 유료 전환 후에도 안전장치로 그대로 쓰인다.
- 필요 API 키: Gemini (+ 향후 OpenAI 1개). Claude는 오디오 입력 미지원이라 불필요.

## 6. 컴포넌트별 수정 계획

| 컴포넌트 | 수정 |
|---|---|
| `alignment.py` | STT 단어 매칭 제거 → 영어 SRT 기반 씬 분할 + 씬 클립 추출기로 전환 |
| `local_stt.py` | 필수 경로에서 제외, 화면 표시용으로 격하 (Demucs 전처리도 선택화) |
| `judge_panel.py` / `providers/gemini.py` | 문화전문가·연출가 입력을 오디오 클립으로 전환, 씬당 1요청 강제, "들은 한국어 병기·불확실 시 보류" 프롬프트, 429 백오프 |
| 신규 `consensus.py` | 후보 씬 블라인드 재청취 + 의미 수준 합의 판정 (합의/폐기/보류) |
| `verdict.py` | 확정 지적만 입력, 판단 보류 커버리지 리포트 추가 |
| 프론트엔드 | 지적 카드에 "들은 한국어" 병기, 합의 수준 배지, 보류 구간 섹션 |

구현 순서: ① 씬 분할+클립 추출 → ② 페르소나 오디오 전환 → ③ 합의 필터 →
④ 판정·프론트 반영 → ⑤ (키 확보 시) 확정층 GPT 교체.
