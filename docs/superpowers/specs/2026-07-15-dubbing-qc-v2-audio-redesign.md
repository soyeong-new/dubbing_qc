# AETHER v2 — 오디오/음성 QC 확장 재설계

- 날짜: 2026-07-15
- 상태: 브레인스토밍 완료, 사용자 승인 대기
- 대상 저장소: dubbing_qc
- 관계: [2026-07-14 v1 설계](2026-07-14-dubbing-qc-redesign-design.md)를 대체하지 않고 확장한다.
  v1은 그 자리에서 계속 확장되는 코드베이스이며(별도 v2 코드베이스 아님), 이 문서는
  v1의 결정에 얽매이지 않고 "지금 아는 것을 기준으로 최선"을 다시 설계한 결과다.

## 1. 이 재설계가 나온 배경

v1(2026-07-14 설계, 이미 구현·운영 중)은 텍스트 로컬라이제이션 QC(3-페르소나 LLM 패널
+ 5축 MOS)를 완성했다. 그런데 실제 운영 중 다음이 드러났다:

- **오디오/음성 차원이 통째로 비어 있었다.** v1의 5축 MOS 중 "음질"·"감정 표현"·
  "싱크 정확도"는 이름만 있었지 텍스트 기반 프록시(발화속도 계산 등)나 미연동 상태
  (`voice_qc.py`)로 남아 있었다.
- **장편 실사용 시 STT가 청크 없이 통째로 호출되어 실패**하고, **Gemini 무료 티어
  요청 한도(일 20건)에 페르소나 패널이 손쉽게 도달**하는 것을 실제로 확인했다.
- 이웃 프로젝트(`AI_Dubbing)auto_proj`, TTS 기반 더빙 자동화 파이프라인)가 화자 분리
  (pyannote)·오디오 감정 인식(wav2vec2 SER)·텍스트 감정 분류(Gemini)를 결합하는 실제
  검증된 패턴을 이미 갖고 있어, 참고할 구체적 사례가 생겼다.

## 2. 스코프 결정: 이웃 프로젝트와의 관계

`AI_Dubbing)auto_proj`(더빙 생산 파이프라인: 화자분리→감정라벨→TTS→믹스)와 이
QC 도구는 **지금 통합하지 않는다.** 두 프로젝트 모두 미완성이고, auto_proj는 후반
작업이 별도로 남아 있어 하나로 끝나지 않는다. 무리하게 지금 연결하면 두 프로젝트의
릴리스 주기가 묶여 스코프가 폭발한다.

**결정:** 완전 분리 유지. 이 문서는 auto_proj 연동을 전제하지 않고, QC가 필요한
모든 입력(화자 분리, 오디오 등)을 **독립적으로 직접 생성**하도록 설계한다. 나중에
두 프로젝트가 모두 완성되면, 데이터 계약(포맷)만 맞춰 연결하는 옵션은 열어두되
지금 설계에는 포함하지 않는다.

## 3. 핵심 철학: "보존" 프레임

v1은 "더빙이 잘 만들어졌는가"를 텍스트 로컬라이제이션에 한해 물었다. v2는 이 프레임을
오디오까지 확장하되, 기준점을 **원본 텍스트가 아니라 원본의 실제 연기(오디오)**로
둔다.

> 대사 텍스트만으로 "이 장면은 화난 내용이니 화나게 연기해야 한다"고 추론하면 틀릴
> 수 있다 — 원본 배우가 그 장면을 실제로 평조로 연기했을 수도 있다. 진짜 기준은
> 원본이 실제로 어떻게 연기했는가이지, 대사 내용에서 추론한 "기대값"이 아니다.

이 원칙은 특히 **감정 표현** 축의 설계를 좌우한다 (§5.2).

## 4. 입력

| 입력 | 필수 여부 | 용도 | v1 대비 변경 |
|---|---|---|---|
| 한국어 원본 오디오 | 필수 | STT 폴백(한국어 SRT 없을 때) + 감정 표현 축 비교 기준 | 유지 |
| 한국어 SRT | 선택 | 있으면 STT 대신 우선 사용 | 유지 |
| 영어 SRT | 필수 | 더빙 대본 텍스트 소스 | 유지 |
| 영어 다이얼로그 스템 | 필수 | 음질·싱크·억양 축 전부의 근거. **분리된 상태로 제공 확인됨** | 유지 (Demucs 등 자체 음원분리 불필요) |
| 한국어 원본 영상 | **불필요** | 어떤 축에도 쓰이지 않음 | **v1 대비 제거** |

영상이 완전히 빠지는 이유: 이번 재설계로 확정된 7축 중 시각 정보(입모양·표정)를
쓰는 항목이 하나도 없다. "싱크 정확도"는 원본↔더빙 **타임코드** 비교로 재정의되어
영상 없이 계산된다 (§5.3).

## 5. MOS 7축 상세 정의

v1의 5축에 2개(억양 적합성, 민감어·욕설)를 추가한다. 축마다 "무엇을 재는지"와
"무엇으로 재는지(기술)"를 명시한다.

### 5.1 언어 적합성 (Linguistic Fit) — v1 재사용

- **측정:** 번역 뉘앙스, 문화적 자연스러움 (직역 오류, 호칭·관용구 현지화)
- **기술:** LLM 텍스트 — 한국 문화·언어 전문가 페르소나 + 영어 원어민 페르소나가
  원문↔번역 대조 (v1의 3-페르소나 패널 그대로)
- **오디오 불필요**

### 5.2 감정 표현 (Emotion Delivery) — v1에서 확장

- **측정:** 이 장면 상황에 맞는 연기를 하고 있는가 — 단, 텍스트에서 추론한 "기대
  감정"이 아니라 **원본이 실제로 연기한 방식**을 기준으로 판단 (§3의 보존 프레임)
- **기술:** LLM 오디오 — 더빙 연출가 페르소나가 **원본 한국어 클립 + 더빙 영어
  클립을 함께 듣고** 비교 판단
- **v1 대비 변경:** v1의 연출가 페르소나는 더빙 오디오만 들었다(원본 비교 불가능
  했음). v2는 씬별로 원본 오디오 클립도 함께 전달한다.
- **왜 수치화(피치/에너지 비교)가 아니라 LLM인가:** 이 프로젝트에서 가장 오래
  검토한 지점이다. 절대값 비교는 원본/더빙이 다른 화자·다른 언어라 무효하고,
  씬 내 상대적 순위 비교는 씬 전체가 통째로 밋밋해지는 경우를 못 잡는 구조적
  맹점이 있다. 두 시도 모두 결함이 확인되어, 이 축은 수치가 아니라 LLM이 두 클립을
  들으며 직접 비교하는 것으로 최종 결정한다. 비용은 별도 패스를 추가하는 게 아니라
  **기존 3-페르소나 호출 중 연출가 페르소나의 입력만 확장**하는 것이므로 신규
  API 호출이 늘지 않는다.

### 5.3 싱크 정확도 (Sync Accuracy) — 재정의 (영상 → 타임코드)

- **측정:** 원본과 더빙의 발화 타이밍이 얼마나 어긋나는가 (v1의 "입모양 동기화"
  정의를 폐기하고 재정의)
- **기술:** 순수 산술 — 정렬(Alignment) 단계에서 이미 계산된 `AlignedPair`의
  (한국어 start/end) vs (더빙 start/end) 격차 계산. 영상도 모델도 불필요.
- **v1 대비 변경:** v1 설계 문서는 이 축을 "입 모양, 장면 전환 타이밍"으로 정의했으나
  영상 입력이 없으므로 애초에 측정 불가능했다. v2는 이미 갖고 있는 정렬 데이터로
  실제로 계산 가능한 정의로 교체한다.

### 5.4 자연스러움 (Naturalness) — v1 재사용

- **측정:** 영어 억양·말 흐름이 원어민에게 자연스러운가 (번역투 여부)
- **기술:** LLM 텍스트 — 영어 원어민 페르소나 (v1 그대로)
- **주의:** 이 축의 "억양"은 "발화가 매끄럽게 들리는가"라는 **품질** 판단이다.
  §5.6의 "억양 적합성"(미국식 표준 부합 여부, **동일성** 판단)과는 다른 질문이며
  혼동하지 않도록 리포트에서 별도 표시한다.

### 5.5 음질 (Audio Quality) — v1 재사용 (다이얼로그 스템 필요)

- **측정:** 클리핑, 잡음(SNR), 명료도
- **기술:** 순수 신호 계산, 모델 불필요
  - 클리핑: 샘플 값이 최대치(±32700/32768, 16bit 기준) 근접 비율
  - 잡음(SNR): 100ms 단위 RMS 에너지, 조용한 구간 대비 발화 구간 에너지 비율(dB)
  - 명료도: SNR + 드롭아웃(대사 구간 내 갑작스런 무음) 종합
- **전제조건:** 이 계산은 **배경음악·효과음이 섞이지 않은 분리된 다이얼로그 스템**을
  전제로 한다. 최종 합본(믹스)에서는 SNR·드롭아웃 계산의 기준점(조용한 구간)이
  사라져 무효화된다. 사용자가 다이얼로그 스템을 항상 제공 가능함을 확인했으므로,
  Demucs 등 자체 음원 분리 로직은 스코프에서 제외한다.

### 5.6 억양 적합성 (Accent Conformance) — 신규, v1에 없던 축

- **측정:** 모든 더빙 대사가 목표 표준 억양(미국식, General American)에 부합하는가
- **기술:** 악센트 분류 모델(예: SpeechBrain CommonAccent 계열) — **세그먼트
  하나하나를 독립적으로** "미국식 / 비-미국식" 분류. 확신도 낮거나 비-미국식으로
  분류되면 플래그.
- **설계 변천 기록 (왜 이 형태로 확정됐는지):**
  1. 최초 제안: 화자 임베딩으로 "같은 캐릭터가 씬마다 억양이 일관되는가"(동일성
     비교) — 캐릭터별 그룹핑(화자 분리 또는 SRT 화자 표기)이 전제조건.
  2. 문제 제기: 화자 분리는 감정 변동으로 같은 캐릭터를 잘못 둘로 쪼개거나, 비슷한
     목소리의 다른 캐릭터를 하나로 잘못 합칠 수 있어 — 특히 후자(오합침)가
     "이 캐릭터 억양이 씬마다 다르다"는 **가짜 경보**를 양산할 위험이 큼.
  3. 최종 결정: "캐릭터별 일관성"이 아니라 "**전체 세그먼트가 고정된 표준(미국식)에
     부합하는가**"로 문제를 재정의. 이러면 캐릭터 그룹핑 자체가 필요 없어져
     화자 분리 의존성과 그 오류 가능성이 전부 사라진다. 각 세그먼트는 서로 독립적으로
     판단되므로, 한 세그먼트의 오분류가 다른 세그먼트 판단에 영향을 주지 않는다.
- **필요 모델:** 악센트 분류기 하나만. 화자 임베딩/화자 분리(pyannote)는 **이
  축에서 더 이상 필요하지 않다.**

### 5.7 민감어·욕설 (Sensitive Content) — 신규, v1에 없던 축

- **측정:** 영어권 수용성 전반(인종/성/종교/정치 등 민감 주제) + 심의·등급 관점
  (욕설 수위, 폭력적 표현)에서 문제가 될 수 있는 대사가 있는가
- **기술:** 텍스트 전용, 오디오 불필요
  1. 금칙어/민감어 사전으로 1차 결정론적 필터 (명백한 욕설·슬러 등)
  2. 사전에 안 걸리는 애매한 표현(암시적 차별, 맥락상 문제 되는 표현)은 LLM이
     대사 텍스트를 읽고 판단
- **결과물 표시:** 5축 MOS 점수에 녹이지 않고 **별도의 "콘텐츠 플래그" 섹션**으로
  리포트에 분리 표시한다 (이유: 품질 점수가 아니라 "문제 있음/없음 + 등급 영향"의
  성격이라 점수화하면 정보가 손실됨). 플래그가 하나라도 있으면 판정 상태와 무관하게
  검수자 확인 필수로 표시.

## 6. 파이프라인 데이터 흐름

```
입력: 한국어 원본 오디오 + (선택)한국어 SRT + 영어 SRT + 영어 다이얼로그 스템
                        │
① Ingest: 한국어(SRT 우선/STT 폴백) + 영어(SRT 고정) 텍스트 세그먼트 생성
② Alignment: 한↔영 타임코드 정렬 + 씬 배정               [v1 재사용]
                        │
        ┌───────────────┴───────────────────────┐
   [A. 결정론적/모델 체크]                  [B. LLM Judge 패널 — 씬 단위]
   A1. 음질: 클리핑/SNR/드롭아웃              B1. 언어 적합성 (문화전문가+원어민)
      (다이얼로그 스템에서 직접)              B2. 자연스러움 (원어민)
   A2. 싱크 정확도: 정렬 데이터의             B3. 감정 표현 (연출가 +
      한↔영 타임코드 격차                        원본·더빙 오디오 클립 동시 청취)
   A3. 억양 적합성: 세그먼트별
      악센트 분류 (독립 판단, 그룹핑 불요)
                        │
   A4. 민감어·욕설: 더빙 대본에 사전 1차 필터 + LLM 정밀 판단 (텍스트 전용)
                        └───────────────┬───────────────────────┘
                            융합 → MOS 7축 + 콘텐츠 플래그 섹션
                                        │
                          대시보드 + 수정 지시서 + 검수 피드백 루프
```

## 7. 컴포넌트 구조 — v1 재사용 vs 신규/변경

| 상태 | 컴포넌트 | 비고 |
|---|---|---|
| **재사용** | `ingest.py` (SRT우선/STT폴백), `alignment.py`, 프로바이더 추상화(`providers/`) | 변경 없음 |
| **재사용** | 언어 적합성·자연스러움 페르소나, 3-페르소나 패널 구조(`judge_panel.py`) | 변경 없음 |
| **재사용** | 클리핑/SNR/드롭아웃 계산(`rule_checks.py`) | 변경 없음 — 다이얼로그 스템 전제 유지 |
| **재사용** | 5축 MOS/판정 엔진 골격(`verdict.py`), 피드백 JSONL 루프 | 7축으로 확장, 판정 로직 자체는 동일 패턴 |
| **변경** | 연출가 페르소나 (`judge_panel.py` 내 director persona) | 입력에 원본 오디오 클립 추가 |
| **변경** | 싱크 정확도 계산 | "입모양" 정의 폐기, 타임코드 격차 계산으로 교체 (신규 함수, 로직은 간단) |
| **신규** | 악센트 분류 모듈 | 세그먼트별 독립 분류, 캐릭터 그룹핑 없음 |
| **신규** | 민감어·욕설 체크 모듈 | 사전 파일 + LLM 텍스트 패스 |
| **v1에서 그대로 미연동 유지** | `context.py`, `voice_qc.py` | 이번 재설계로도 여전히 호출되지 않음. 초기 프로토타입의 잔여 파일 — 삭제하지 않고 보존 (실모델 연동 시 재검토) |
| **폐기 (설계에서 제외)** | 화자 분리(diarization) 기반 캐릭터 그룹핑, 화자 임베딩 기반 억양 "일관성" 비교, Demucs 음원 분리 | §5.6의 설계 변천 참조. 다이얼로그 스템 직접 제공 확인 + 억양 축 재정의로 전부 불필요해짐 |

**이번 재설계에서 유일하게 새로 추가되는 무거운 ML 의존성은 악센트 분류 모델
하나뿐이다.** 화자 임베딩·화자 분리·음원 분리는 검토했지만 전부 불필요한 것으로
결론 내렸다.

## 8. 비용·정확도 완화 전략

7축 중 LLM이 필요한 것은 3개(언어 적합성/자연스러움/감정 표현)뿐이며, 나머지 4개
(음질/싱크/억양 적합성/민감어 1차 필터)는 결정론적 계산 또는 독립 분류 모델로
처리해 LLM 호출 자체를 줄인다. LLM이 필요한 3축도 씬 단위 배치 호출(줄 단위 아님,
v1 패턴 유지)이며, **AI 판정은 전부 가판정 — 검수자의 승인/반려로 최종 확정**되므로
LLM 판단의 불완전한 정확도가 그대로 최종 리포트에 실리지 않는다. 추가 비용 절감이
필요해지면 §9(로드맵, v1 설계 문서 §11 참조)의 저비용 스크리너 도입을 검토한다.

## 9. 오류 처리 (v1 원칙 유지)

- STT 실패/저신뢰: 정렬 신뢰도 저하 finding으로 표면화, 파이프라인 계속 진행
- LLM 호출 실패: 1회 재시도 후 실패 시 해당 씬 건너뜀 (v1과 동일)
- 악센트 분류 실패(모델 로드 오류 등): 해당 세그먼트를 "미분류"로 표시, 전체 축
  판정에는 미분류 세그먼트를 제외하고 계산 (침묵 처리하지 않고 리포트에 "N개
  세그먼트 미분류" 명시)
- mock 자동 폴백 금지 원칙 그대로 유지 (v1 §4)

## 10. 범위 밖 (명시적 제외)

- `AI_Dubbing)auto_proj`와의 실제 연동 (§2 — 계약은 나중에, 지금은 완전 분리)
- 화자 분리/화자 임베딩 기반 캐릭터별 억양 일관성 체크 (§5.6 — 재정의로 대체됨)
- 다이얼로그 스템이 없는 경우의 자동 음원 분리(Demucs) — 스템 항상 제공 확인됨
- 영상 기반 립싱크 분석 — 어떤 축에도 불필요한 것으로 확정
- xCOMET/MetricX 텍스트 스크리너, 로컬 LoRA 모델 (v1 설계 문서 §11 로드맵 그대로 유지, 이번 재설계 범위 아님)
